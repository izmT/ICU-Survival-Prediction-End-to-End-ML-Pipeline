# ICU Mortality Prediction — End-to-End ML Pipeline on Imbalanced Clinical Data

> **Binary classification of in-hospital patient mortality using supervised machine learning on high-dimensional synthetic clinical data.**


## Project Overview

This project tackles a real-world challenge in medical AI: predicting whether a patient will die during their hospital stay, given their clinical record at admission. The dataset is synthetic but clinically realistic, generated via a Wasserstein GAN from real ICU data (MIMIC) to preserve patient confidentiality.

The core challenge is **class imbalance**: only 3.5% of patients die, making standard accuracy meaningless. The project implements a complete, production-grade ML pipeline from raw data to final submission.

**Best result: Balanced Accuracy = 0.672 (CV 5-fold) — GaussianNB + PCA(n=10) + RandomOverSampler**

---

## Challenge

- **Source**: [Codalab MediChal Challenge](https://codalab.lisn.upsaclay.fr/competitions/3739)
- **Task**: Binary classification — predict `DIED` (0 = survival, 1 = death)
- **Metric**: `balanced_accuracy_score` (average recall per class)
- **Dataset**: Synthetic medical records generated by Wasserstein GAN

## Pipeline

```
Raw Data (341 vars)
      ↓
Feature Selection (point-biserial correlation)
      ↓ 177 variables removed
Clean Data (164 vars)
      ↓
Rare Category Merging (ICU, INSURANCE)
      ↓
One-Hot Encoding + Median Imputation
      ↓
Stratified Train/Valid Split (80/20)
      ↓
RandomOverSampler (train only — no leakage)
      ↓
StandardScaler → PCA(n=10) → GaussianNB
      ↓
GridSearchCV (5-fold, balanced_accuracy)
      ↓
Evaluation + Submission
```

## Feature Selection

A key contribution of this project is the **target-oriented feature selection** using point-biserial correlation between each variable and `DIED`.

### Variables removed (177 total):

| Category | Count | Reason |
|---|---|---|
| Constant columns (always 0) | 173 | Zero variance — no information |
| Categorical (no signal, no clinical basis) | 4 | RELIGION, MARITAL_STATUS, ETHNICITY, LANGUAGE |

### Categorical variables kept:

| Variable | Correlation | p-value | Justification |
|---|---|---|---|
| GENDER | 0.020 | ≈0 | Physiological differences M/F in ICU |
| ICU | 0.014 | 0.0001 | ICU unit reflects case severity |
| INSURANCE | 0.001 | 0.792 | Clinical justification: proxy for SES → care access (USA/MIMIC context) |

### Numerical/binary variables:
All retained — clinically meaningful (comorbidities, vital signs). Low individual correlations are expected on synthetic data; PCA handles dimensionality reduction downstream.

## Results

### Model comparison (with RandomOverSampler + PCA n=20)

| Model | Balanced Acc. | Recall | Precision |
|---|---|---|---|
| Logistic Regression | 0.696 | 0.689 | ~0.08 |
| **Gaussian NB** | 0.655 | 0.542 | ~0.08 |
| KNN (k=5) | 0.562 | 0.199 | ~0.09 |
| Random Forest (n=20) | 0.501 | 0.002 | 0.333 |

### GridSearch — optimal PCA components

| n_components | Balanced Acc. (CV) | Std |
|---|---|---|
| **10** | **0.672** | 0.001 |
| 20 | 0.656 | 0.002 |
| 5 | 0.652 | 0.003 |

### CV comparison: GaussianNB vs LogisticRegression (n=10)

| Model | Mean Bal. Acc. | Std |
|---|---|---|
| **GaussianNB** | **0.672** | 0.001 |
| LogisticRegression | 0.666 | 0.002 |

### Final evaluation — GaussianNB + PCA(n=10) + ROS

| Metric | Value |
|---|---|
| Balanced Accuracy | ~0.667 |
| Recall (deaths) | 0.635 |
| Precision | 0.071 |
| F1-Score | 0.128 |
| AUC-ROC | 0.723 |

### Confusion Matrix

|  | Predicted Survival | Predicted Death |
|---|---|---|
| **Real Survival** | VN = 10,789 ✓ | FP = 4,652 |
| **Real Death** | FN = 204 ✗ | VP = 355 ✓ |

355 deaths detected out of 559 — 204 missed (FN: most critical clinical error).

### ROS vs SMOTE

| Method | Balanced Acc. (CV) | Recall |
|---|---|---|
| **RandomOverSampler** | **0.672** ± 0.001 | — |
| SMOTE (integrated CV) | 0.637 ± 0.017 | — |

ROS outperforms SMOTE with GaussianNB — SMOTE interpolation is poorly suited to binary medical variables and disrupts Gaussian assumptions.

## Why Balanced Accuracy?

With 96.5% survivors, a naive classifier predicting "survival" always achieves 96.5% standard accuracy — but detects 0% of deaths. Balanced accuracy is the mean recall per class:

```
balanced_accuracy = (recall_class_0 + recall_class_1) / 2
```

It equals 0.5 for a random classifier and 1.0 for a perfect one, regardless of class imbalance.

## Key Design Decisions

| Decision | Rationale |
|---|---|
| One-Hot over Label Encoding | No artificial ordering between categories |
| Stratified 80/20 split | Preserves 3.5% death ratio in both sets |
| ROS on train only | Prevents data leakage into validation |
| PCA after scaling | StandardScaler mandatory before PCA |
| GridSearch on 30k sample | Memory constraint (full dataset crashes kernel) |
| GaussianNB as final model | Best CV score (0.672) + fastest training |
| INSURANCE kept despite low correlation | Clinical justification: SES proxy in US healthcare |

---

## Limitations

- Feature selection performed on **synthetic (GAN-generated) data** — correlations may not reflect real MIMIC patterns
- GridSearch limited to 30k patients due to memory constraints
- SMOTE underperforms with GaussianNB — would benefit from tree-based models
- Precision remains low (~0.07) — inherent to extreme class imbalance
